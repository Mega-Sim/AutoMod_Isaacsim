# AutoMod pm.asy to Isaac Sim JSON Format

## Overview

This document describes the JSON format generated by converting AutoMod Path Manager (.asy) files to Isaac Sim layout definitions.

The converter parses AutoMod's path, control point, and vehicle definitions and converts them to a structured JSON format suitable for Isaac Sim digital twin applications.

## File Generation

### Command

```bash
python3 scripts/pm_asy_to_json.py \
  --input data/raw/basic_model/model.arc/pm.asy \
  --output generated/basic_model_layout.json \
  --snap-tolerance-mm 1.0 \
  --arc-chord-error-mm 5.0
```

### Options

- `--input`: Path to AutoMod pm.asy file
- `--output`: Output JSON file path
- `--snap-tolerance-mm`: Tolerance for snapping nearby nodes (default: 1.0)
- `--arc-chord-error-mm`: Maximum chord error for arc polyline approximation (default: 5.0)

## JSON Structure

### Top-Level Keys

```json
{
  "metadata": { ... },
  "system_parameters": { ... },
  "paths": [ ... ],
  "nodes": [ ... ],
  "edges": [ ... ],
  "control_points": [ ... ],
  "stations": [ ... ],
  "name_lists": [ ... ],
  "vehicles": [ ... ],
  "validation": { ... }
}
```

## Detailed Schema

### metadata

Contains conversion and file metadata.

```json
{
  "source_file": "pm.asy",
  "source_units": "Millimeters",
  "output_units": "Meters",
  "up_axis": "Z",
  "snap_tolerance_mm": 1.0,
  "arc_chord_error_mm": 5.0
}
```

### system_parameters

Contains AutoMod system-level parameters extracted from SYSDEF, UNITS, etc.

```json
{
  "timeout": 60,
  "Seconds": "confname",
  ...
}
```

### paths

Array of source paths (GPATH records) with geometry.

#### Line Path Example

```json
{
  "id": "path100338",
  "name": "path100338",
  "type": "DefaultGuidePath",
  "geometry_type": "line",
  "length_mm": 207247.892,
  "length_m": 207.248,
  "geometry": {
    "type": "line",
    "begin": {"x": -102104.668, "y": 110843.164},
    "end": {"x": 105143.224, "y": 110843.164}
  }
}
```

#### Arc Path Example

```json
{
  "id": "path100901",
  "name": "path100901",
  "type": "DefaultGuidePath",
  "geometry_type": "arc",
  "length_mm": 353.429,
  "length_m": 0.353,
  "geometry": {
    "type": "arc",
    "center": {"x": 90989.408, "y": 111293.152},
    "begin": {"x": 90989.443, "y": 110843.161},
    "end": {"x": 91307.624, "y": 110974.984},
    "radius_mm": 450.0,
    "sweep_angle_deg": 45.0,
    "sweep_angle_rad": 0.7854
  }
}
```

### nodes

Array of graph nodes representing path endpoints and control point locations.

```json
{
  "id": "node_000000",
  "position_mm": {"x": 0.0, "y": 0.0},
  "position_m": {"x": 0.0, "y": 0.0},
  "source_paths": ["path100"],
  "control_points": ["cp_Park_1"]
}
```

**Snapping**: Nearby endpoints (within snap tolerance) are merged into single nodes.

### edges

Array of directed edges connecting nodes, derived from GPATH segments split by control points.

```json
{
  "id": "edge_path100_0000",
  "from_node_id": "node_000001",
  "to_node_id": "node_000002",
  "source_path_id": "path100",
  "source_distance_start_mm": 0.0,
  "source_distance_end_mm": 1000.0,
  "length_m": 1.0,
  "length_mm": 1000.0,
  "geometry_type": "line",
  "direction": "forward",
  "one_way": true,
  "polyline_m": [
    [0.0, 0.0],
    [1.0, 0.0]
  ]
}
```

**Polyline**: For arc edges, polyline points are computed using chord error approximation.

### control_points

Array of all control points (CPOINT records) with computed positions.

```json
{
  "name": "cp_Park_1",
  "type": "DefaultControlPoint",
  "path_id": "path220",
  "distance_mm": 15992.857,
  "position_mm": {"x": 12345.0, "y": 67890.0},
  "position_m": {"x": 12.345, "y": 67.890},
  "raw_attributes": { ... }
}
```

### stations

Filtered subset of control_points classified as operational stations.

Currently empty in basic_model (requires classification rules configuration).

### name_lists

Groups of control points defined via NAMELST records.

```json
{
  "name": "NAME_ALL",
  "members": ["cp_A01001", "cp_A01002", ...],
  "raw_attributes": { ... }
}
```

### vehicles

Array of vehicle type definitions (AGVSVEH records).

```json
{
  "type": "EVL",
  "numveh": 150,
  "start": "Random",
  "raw_attributes": { ... }
}
```

### validation

Validation results and statistics.

```json
{
  "errors": [],
  "warnings": [],
  "counts": {
    "gpaths": 1637,
    "line_gpaths": 641,
    "arc_gpaths": 996,
    "cpoints": 468,
    "cpoint_types": 8,
    "nodes": 2524,
    "edges": 2101,
    "stations": 0,
    "vehicles": 3
  }
}
```

## Coordinate Systems

### AutoMod → Isaac Sim Conversion

- **Source**: AutoMod XY plane in millimeters
- **Target**: Isaac Sim XY plane in meters (Z-up coordinate system)
- **Conversion**: `x_m = x_mm / 1000`, `y_m = y_mm / 1000`, `z_m = 0.0` (default)

All coordinates in the JSON are provided in both source (mm) and target (m) units.

## Arc Geometry

### Raw Format

AutoMod stores arc sweeps in 0.1-degree units. For example, `angle 449.998779296892` represents a ~45-degree sweep.

```
angle_degrees = angle_tenths / 10.0
```

### Sweep Direction

- **Positive sweep**: Counterclockwise rotation from begin point around center
- **Negative sweep**: Clockwise rotation

### End Point Calculation

The end point is computed by rotating the begin point around the center by the sweep angle:

```
end = rotate(begin, sweep_radians, center)
```

### Arc Length

```
length = radius * abs(sweep_radians)
```

### Polyline Approximation

For visualization in Isaac Sim, arc paths are approximated as polylines using maximum chord error:

```
max_chord_error = arc_chord_error_mm  # CLI parameter
num_segments = ceil(|sweep_radians| / max_angle_per_segment)
```

## Path Splitting by Control Points

When control points lie on a path, that path is split into multiple edges:

```
path100 with control points at distance 1000 and 3000:
  - edge: node_0 → node_1 (distance 0 → 1000)
  - edge: node_1 → node_2 (distance 1000 → 3000)
  - edge: node_2 → node_3 (distance 3000 → path_length)
```

Control points at identical distances are merged into single nodes.

## Data Validation

### Checks Performed

1. All GPATH records successfully parsed (line or arc geometry)
2. All CPOINT references valid source paths
3. Control point distances within path lengths
4. All edge lengths positive
5. All node references valid
6. All coordinates finite
7. No duplicate node locations (within snap tolerance)
8. All vehicles and name_lists complete

### Result Fields

- `validation.errors`: Critical issues (should be empty for valid files)
- `validation.warnings`: Non-critical issues (e.g., out-of-range control points)
- `validation.counts`: Statistics on parsed records

## Usage in Isaac Sim

### Key Fields for Isaac Sim USD Generation

1. **Paths & Geometry**:
   - `paths[].geometry_type`: Determines curve type (line or arc)
   - `paths[].geometry`: For arcs, includes center, radius, sweep angle
   - `edges[].polyline_m`: Polyline approximation for visualization

2. **Navigation**:
   - `edges[].from_node_id`, `to_node_id`: Graph connectivity
   - `edges[].length_m`: Distance for path planning
   - `edges[].direction`, `one_way`: Direction constraints

3. **Stations**:
   - `control_points[].position_m`: Station placement
   - `control_points[].tangent_yaw_rad`: OHT orientation at station

4. **Vehicles**:
   - `vehicles[].numveh`: Fleet size
   - `vehicles[].start`: Initial location

## Common Issues

### Zero-Length Edges

Occur when multiple control points have identical distances. These are filtered during JSON generation.

### Control Points Outside Path Bounds

Warnings are issued but control points are still included. Verify with source file.

### Node Snap Tolerance

Default 1.0 mm tolerance may need adjustment for:
- Very small layouts: reduce tolerance
- Very large layouts: increase tolerance to reduce node count

## Example: Converting and Validating

```bash
# Convert
python3 scripts/pm_asy_to_json.py \
  --input data/raw/basic_model/model.arc/pm.asy \
  --output generated/basic_model_layout.json

# Validate
python3 -m unittest discover -s tests -v

# Inspect
python3 -c "
import json
with open('generated/basic_model_layout.json') as f:
    data = json.load(f)
print(f'Paths: {len(data[\"paths\"])}')
print(f'Nodes: {len(data[\"nodes\"])}')
print(f'Edges: {len(data[\"edges\"])}')
"
```
